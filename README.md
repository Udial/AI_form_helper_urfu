# План разработки: AI-ассистент для генерации студенческих заявок
## Общая цель
Создать демо-веб-приложение, в котором пользователь:
1. Вводит идею проекта (текстом или голосом)
2. Открывает AI-ассистента (pop-up)
3. Получает сгенерированную заявку
4. Вставляет результат в форму
---
# 1. Frontend разработчик (основная страница и форма)
## Цель
Реализовать страницу заявки и интеграцию с ассистентом
## Задачи
### 1. Базовая структура страницы
- Создать layout страницы
- Разделить на 2 колонки:
  - форма
  - панель ассистента (кнопка)
- Добавить базовую адаптивность
**Результат:** готовая страница с разметкой
---
### 2. Форма заявки
- Поля:
  - Название проекта
  - Описание
  - Цели
- Реализовать controlled inputs
- Добавить базовую валидацию:
  - обязательные поля
  - минимальная длина
**Результат:** форма корректно работает
---
### 3. Кнопка ассистента
- Разместить кнопку “AI помощник”
- Реализовать открытие pop-up
**Результат:** открывается окно ассистента
---
### 4. Интеграция с API
- Отправка запроса на backend (`/generate`)
- Получение ответа
**Результат:** данные приходят с сервера
---
### 5. Вставка результата
- Кнопка “Вставить в форму”
- Автозаполнение полей формы
**Результат:** AI → форма
---

# 3. Backend разработчик (API)
## Цель
Реализовать серверную часть и API
## Задачи
### 1. Поднять сервер
- Выбрать стек (Express / FastAPI / Django)
- Создать базовую структуру
**Результат:** сервер работает
---
### 2. Endpoint генерации
- POST `/generate`
- Вход:
```json
{ "idea": "..." }
```
**Результат:** endpoint принимает запрос
---
### 3. Интеграция с AI

- Вызов LLM API
- Передача prompt

**Результат:** возвращается сгенерированный текст
---
###4. Формат ответа

- Вернуть структурированный JSON:
```json
{
  "title": "...",
  "description": "...",
  "goals": "..."
}
```
**Результат:** фронтенд может парсить ответ
---
### 5. Endpoint для аудио

- POST /transcribe
- Приём аудиофайла

**Результат:** аудио преобразуется в текст
---
### 6. Авторизация (минимальная)

- Заглушка логина
- Токен (JWT или сессия)

**Результат:** ограниченный доступ
---

# Общий флоу системы

### 1. Пользователь вводит идею проекта
### 2. Frontend → POST /generate
### 3. Backend → AI API
### 4. AI → возвращает JSON
### 5. Frontend отображает результат в чате
### 6. Пользователь нажимает “Вставить”
### 7. Данные попадают в форму
